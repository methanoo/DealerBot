    Sistema di controllo infrarossi — Protocollo NEC su ATtiny85

    NEC 38kHz ATtiny85 Arduino ISP IRremote v4.x C++ / Arduino
    !
    Panoramica del Sistema

    Il sistema DealerBot utilizza telecomandi IR (protocollo NEC) per identificare ogni giocatore e attivare la distribuzione automatica delle carte.

    Architettura del Sistema

    Pinout ATtiny85
    ┌──────┐ PB5 (RST) ─┤1 8├─ VCC (5V/3.3V) PB3 ─┤2 7├─ PB2 (SCK) PB4 ─┤3 6├─ PB1 ← LED IR (Pin 1) GND ─┤4 5├─ PB0 ← BUTTON (Pin 2) └──────┘

    Logica di Funzionamento

    Flusso Operativo

    • 1
      Il giocatore preme il pulsante sul telecomando
    • 2
      L'ATtiny85 invia un codice IR univoco via LED IR
    • 3
      Il ricevitore centrale decodifica il segnale NEC
    • 4
      Viene assegnata la carta corrispondente al giocatore

    Firmware e Programmazione

    Setup programmatore

    Step 1 — Prepara l'Arduino UNO come programmatore:

    • Collega Arduino UNO al PC via USB
    • Carica lo sketch ArduinoISP (File → Esempi → ArduinoISP)
    • Vai in Strumenti → Programmatore → seleziona Arduino as ISP

    Schema di collegamento Arduino UNO ↔ ATtiny85

    Arduino UNOATtiny85Descrizione
    5VVCCAlimentazione
    GNDGNDMassa
    Pin 10RESETReset programmatore
    Pin 11MOSIDati verso chip
    Pin 12MISODati dal chip
    Pin 13SCKClock SPI

    ⚠️
    Condensatore anti-reset

    Aggiungi un condensatore da 10µF tra RESET e GND di Arduino UNO per evitare il reset automatico durante la programmazione.

    Configurazione IDE

    Vai in Strumenti e configura i seguenti parametri:

    SchedaATtiny85
    ClockInternal 8 MHz
    ProgrammerArduino as ISP

    !
    Importante

    Clicca su Strumenti → Burn Bootloader per impostare i fuses, e infine Sketch → Carica con Programmatore.

    Ora puoi passare alla prossima scheda: Trasmettitore (ATtiny85)

    Funzione

    Invia segnali IR per identificare univocamente il giocatore che preme il tasto.

    ⚠️
    Attenzione — Codici Univoci

    Ogni telecomando deve avere un codice HEX diverso nel parametro command. Usa la tabella di assegnazione per assegnare correttamente i codici.

    Codice sorgente

    #include <Arduino.h>
#include <TinyIRSender.hpp>

#define IR_PIN      1   // PB1 — LED IR (con resistenza da 100Ω)
#define BUTTON_PIN  2   // PB2 — Pulsante (INPUT_PULLUP)

// ┌───────────────────────────────────────┐
// │  TABELLA ASSEGNAZIONE GIOCATORI       │
// │                                       │
// │  Giocatore 1 → 0x0C                   │
// │  Giocatore 2 → 0x18                   │
// │  Giocatore 3 → 0x5E                   │
// │  Giocatore 4 → 0x08                   │
// │  Giocatore 5 → 0x1C                   │
// │  Giocatore 6 → 0x5A                   │
// │  Giocatore 7 → 0x42                   │
// └───────────────────────────────────────┘

void setup() {
    pinMode(BUTTON_PIN, INPUT_PULLUP);
}

void loop() {
    if (digitalRead(BUTTON_PIN) == LOW) {
        sendNEC(1, 0x0C, 0);   // ← modifica 0x0C per ogni giocatore
        delay(500);          
    }
}

    📦
    Libreria richiesta

    Installa IRremote (v4.x) o TinyIRSender dal Library Manager dell'IDE.

    Scopo

    Verifica i codici IR ricevuti durante lo sviluppo e il testing dei telecomandi.

    💡
    Quando usarlo

    Usa questo sketch su un Arduino UNO collegato al PC per leggere i codici di ogni telecomando prima di integrarli nel firmware principale.

    Pinout ricevitore IR

    Modulo IR (es. VS1838B / TSOP4838) │ ├── VCC → 5V ├── GND → GND └── OUT → Pin 11 (Arduino)

    Codice sorgente

    #include <IRremote.hpp>

#define IR_RECEIVE_PIN  11    // Pin dati del modulo VS1838B / TSOP4838

void setup() {
    Serial.begin(9600);
    IrReceiver.begin(IR_RECEIVE_PIN, ENABLE_LED_FEEDBACK);
    Serial.println("=== DealerBot IR Debugger ===");
    Serial.println("In attesa di segnali IR...");
}

void loop() {
    if (IrReceiver.decode()) {
        Serial.print("[ OK ]  Codice ricevuto: 0x");
        Serial.print(IrReceiver.decodedIRData.decodedRawData, HEX);
        Serial.print("  |  Protocollo: ");
        Serial.println(IrReceiver.decodedIRData.protocol);
        IrReceiver.resume();
    }
}
